ANR-分析之基础知识介绍：event log

发表于2024-08-24|更新于2024-10-16|ANR
|阅读量:

上一章介绍了ANR分析基础知识trace信息，trace信息是ANR分析的必要信息之一，里面包含了ANR发生事件，ANR发生时堆栈信息，线程状态等。

本章介绍一下另外一个必要信息：Event log。

下面信息基于Android 14的代码。

Event log分类

  • am相关信息
  • app相关信息
  • dispatcher相关信息
  • server相关信息
  • systemui相关信息
  • vm相关信息

Event log抓取

如果想要获取更详细的信息可以加对应参数

Event log在源码中的信息详情

am

位置：/frameworks/base/services/core/java/com/android/server/am/EventLogTags.logtags

内容：

这是系统预设的log，我们可以根据这些log去分析系统的流程或者问题，

例如：

这个是发生ANR的类型，后面包含详细信息，User，pid，包名，原因等。

了解这些就可以知道每个对应log包含的内容以及含义。

常用的还有：

每个log上面都有对应的解释，后面都包含了详细信息，通过这些信息我们可以确定发生的进程，原因等，了解这些可以让我们比较容易的查找原因，分析流程。

app

位置：/frameworks/base/core/java/android/app/EventLogTags.logtags

内容：

这个log主要是应用端窗口启动或者状态改变的信息。

dispatcher

位置：/frameworks/native/services/inputflinger/dispatcher/EventLogTags.logtags

内容：

这个是我们查看input焦点窗口信息的，通过该log帮助我们查看操作过程中焦点的切换流程，帮助我们分析窗口焦点丢失的ANR原因。

server

位置：/frameworks/base/services/core/java/com/android/server/EventLogTags.logtags

内容：

这个是有关系统server的log信息，里面每行都有详细解释。包含了很多服务的状态。

systemui

位置：/frameworks/base/packages/SystemUI/src/com/android/systemui/EventLogTags.logtags

内容：

vm

位置：/frameworks/base/services/core/java/com/android/server/wm/EventLogTags.logtags

内容：

这个是server端窗口状态的log信息。

由于ANR涉及应用端，系统端，涉及UI，流程，代码逻辑，窗口切换，Input事件分发，系统绘制等各个方面，因此要分析ANR就需要了解应用端流程，以及系统流程，因此学习上面这些log对应的内容也是非常重要的，熟练了解这些内容后，对于我们分析ANR或者其他问题都是非常有帮助的。

当然，要记住这么多log还是有难度的，主要是理解，看到能知道什么含义，如果不熟悉，那知道怎么查找也是有帮助的。

本章我个人解释的东西比较少，主要是里面都有注释和含义，基本都能看懂，更详细的需要去看源码，查看源码中代码流程。
